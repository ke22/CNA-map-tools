## ⏱️ 免費帳號處理時間

由於是免費帳號，處理時間可能較長：

| 文件 | 大小 | 預計處理時間 |
|------|------|------------|
| Level 0 | 8.4 GB | 2-4 小時 |
| Level 1 | 6.4 GB | 1.5-3 小時 |
| Level 2 | 6.6 GB | 1.5-3 小時 |

**總計：約 5-10 小時**（可以同時上傳，但處理是順序的）

## ⚠️ 重要：免費帳號上傳限制

### Mapbox Studio 網頁上傳限制

**限制：** 單個文件 ≤ 300 MB

**您的文件：**
- Level 0: 8.4 GB ❌ **超過限制**
- Level 1: 6.4 GB ❌ **超過限制**
- Level 2: 6.6 GB ❌ **超過限制**

**結論：** ❌ **不能使用網頁上傳！**

---

## ✅ 解決方案：使用 API 上傳

免費帳號可以使用 **Mapbox Uploads API** 上傳大文件（最大 25 GB）。

### 步驟 1: 準備 Token 和用戶名

```bash
# 設置 Token（從 config.js 或獲取）
export MAPBOX_ACCESS_TOKEN=your_token_here

# 記下用戶名（在 Mapbox Studio 中查看）
```

### 步驟 2: 使用 curl 腳本上傳

```bash
./scripts/upload-to-mapbox-curl.sh
```

這個腳本會使用 API 上傳，支持大文件！

### 步驟 3: 等待處理完成

- 在 Mapbox Studio 查看處理狀態
- 處理完成後會顯示為 "Ready"
- 記錄 Tileset ID

### 步驟 4: 測試後繼續

處理完第一個後：
- 測試應用是否正常工作
- 如果成功，繼續上傳其他兩個文件

## 💡 免費帳號建議

### 建議 1: 分批上傳

不要一次上傳所有文件：
1. 先上傳 Level 1 (6.4 GB - 最小)
2. 等待處理完成
3. 測試
4. 再決定是否上傳 Level 0 和 Level 2

### 建議 2: 考慮使用簡化版本

如果您不想等待長時間處理，可以：

```bash
# 使用方案 1：簡化 GeoJSON（更快，無需上傳）
./scripts/optimize-gadm.sh
```

這會生成更小的 GeoJSON 文件（50-400 MB），可以直接使用，無需上傳到 Mapbox。
