# 合并搜索功能分析

## 当前状态

### 区域搜索 (Search Area)
- **位置**：Quick Start 区域
- **功能**：搜索国家/行政区并着色
- **结果**：对整个区域应用颜色

### 标记搜索 (Add Marker)
- **位置**：Markers 区域
- **功能**：搜索地点或输入坐标添加标记
- **结果**：在地图上添加点标记

## 合并选项分析

### 选项 1：保持分开（推荐）✅

**优点：**
- 功能清晰，不会混淆
- 用户明确知道操作结果
- UI 结构更清晰
- 维护更简单

**缺点：**
- 有两个搜索框

### 选项 2：合并为一个搜索框

**实现方式 A：智能识别 + 操作选择**
- 搜索后显示两个选项：着色区域 / 添加标记
- 用户选择操作类型

**实现方式 B：根据模式自动判断**
- 如果当前在"国家"模式 → 着色区域
- 如果当前在"标记"模式 → 添加标记

**优点：**
- 只有一个搜索框
- 界面更简洁

**缺点：**
- 可能造成混淆
- 用户可能不确定操作结果
- 实现更复杂

## 推荐方案

### 方案 1：保持分开（简单清晰）⭐
- 区域搜索：搜索并着色区域
- 标记搜索：搜索并添加标记
- 各司其职，功能明确

### 方案 2：改进区域搜索（折中方案）
- 区域搜索可以识别坐标
- 如果是坐标，提示："检测到坐标，是否添加标记？"
- 提供快捷链接到标记功能

### 方案 3：合并为统一搜索（高级方案）
- 创建一个统一的搜索框
- 搜索结果显示两个操作选项：
  - "着色此区域"
  - "添加标记"
- 用户选择要执行的操作

## 建议

**推荐保持分开**，因为：
1. 两个功能的目的完全不同
2. 用户通常知道自己要做什么
3. 分开更直观，不容易出错

如果需要，可以在区域搜索结果中添加"添加标记"的快捷选项。
