# Shared Context 目录

> 此目录用于跨工作阶段共享上下文信息，记录项目配置、已知问题、技术决策等

## 📁 文件说明

### `known-issues.md`
记录技术债、发现的问题和待办事项
- 技术债（高/中/低优先级）
- 发现和观察
- 已解决的问题
- 待办事项
- 调试技巧

### `port-configuration.md`
端口配置和使用说明
- 端口分配总览
- Map Tool v1 端口配置（8000）
- Map Tool v2 端口配置（8001）
- 端口检查命令
- 同时运行两个项目的说明

## 🎯 使用目的

这个目录的目的是：
1. **跨会话共享**: 在不同工作会话之间保留重要信息
2. **技术决策记录**: 记录重要的技术决策和配置
3. **问题追踪**: 跟踪已知问题和解决方案
4. **快速参考**: 提供快速查找重要信息的场所

## 📝 维护指南

- 当发现新问题或技术债时，更新 `known-issues.md`
- 当配置发生变化时，更新相应的配置文件
- 定期审查和更新文档，确保信息准确
- 重要变更应记录在相应文档的"更新记录"部分
